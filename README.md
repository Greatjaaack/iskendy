# Искенди — табло заказов

Веб-табло готовности заказов для дюрюмной «Искенди». Заказы подтягиваются из
кассы **iiko** и появляются на табло; касса двигает их по статусам, а гость с
телефона (или с телевизора в зале) видит, готов ли его номер. Снимает толпу у
окна в пик.

Проект **отдельный** от аналитического дашборда `iskendy_analytic`: другая
аудитория (гость с улицы, а не владелец), другой жизненный цикл, отдельный
деплой (но данные о заказах табло берёт из iiko через ручку аналитики).

## Модель

- Статус ведётся **на уровне заказа**: `open` (открытый — приехал из iiko, ещё
  не взяли в работу) → `preparing` (готовится) → `ready` (готово) → `served`
  (выдано). «served» снимает заказ с табло, но остаётся в истории.
- **Источник заказов — iiko** (Фаза 1, поллинг): фоновый `iiko_poller` раз в
  ~30 сек дёргает ручку аналитики (`/api/orders/today`) и заводит новые заказы
  как «открытые» (окно свежести — последние 20 мин, дедуп по номеру за день).
  Ручной ввод номера на кассе остаётся как fallback (создаёт сразу «готовится»).
- По каждому заказу хранятся **времена этапов**: приём (`created_at`), готово
  (`ready_at`), выдано (`served_at`). Плюс **журнал событий** (`order_events`) —
  аудит всех переходов.
- Гость видит заказ с «Готовится» (статус «открытый» — внутренний, для кассы).
  Номер показывается такой же, как в iiko (напр. 5221).

## Экраны (маршруты)

- `/` — лендинг-мультиссылка (главная).
- `/board` — гостевое табло: колонки «Готовятся» / «Готово», номера растут вбок
  по 5 в столбик; вверху QR на это же табло (скан с телевизора → на телефон).
- `/tv` — тот же контент крупно, для **телевизора в зале** (подогнан под 1366×768).
- `/staff` — касса: занести заказ, двигать статусы (цвета: 🔵 открытый · 🟠
  готовится · 🟢 готово), счётчики по статусам, журнал событий.
- `/stats` — аналитика (под паролем кассы): выбор дней (день недели), KPI и
  столбчатые графики «заказы по часам» и «среднее время по часам».

Один статичный `index.html` обслуживает все экраны — выбор по URL.

## Стек

- **Backend:** FastAPI + SQLite (stdlib `sqlite3`), JWT-авторизация кассы на
  `hmac`. Фоновые задачи: iiko-поллер, ночной бэкап. Заказы из iiko — через httpx.
- **Frontend:** один статичный `index.html` без сборки (палитра «Bakır» / бренд-
  градиент, инлайн-SVG графики без внешних библиотек). Опрос статуса раз в 5 сек.

## Данные и бэкап

- Всё в SQLite `iskendy.db` (в проде — Docker volume `app-data` → `/data`).
  Таблицы: `orders`, `order_events` (+ неиспользуемые `day_state`/`batch_log`
  с эпохи «партий»).
- **Ночной бэкап** (`backup.py`): раз в день в 03:00 снимает gzip-копию в
  `/data/backups/` с ротацией (последние 30). Скачать копию наружу — `GET
  /api/backup/latest`.

## Запуск в Docker

БД живёт в volume `app-data`. Нужен заполненный `.env` в корне.

```bash
cp .env.example .env        # задать STAFF_PASSWORD (+ IIKO_ORDERS_URL/TOKEN для iiko)
docker compose up -d --build
```

- Лендинг: http://localhost:8080/ · табло: `/board` · ТВ: `/tv` · касса: `/staff`
  · аналитика: `/stats`.

## Запуск локально без Docker

```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
STAFF_PASSWORD=<пароль> uvicorn main:app --reload --port 8080
```

## API

| Метод | Путь | Кто | Что |
|-------|------|-----|-----|
| GET  | `/api/status`        | публично | активные заказы (открытые + готовятся + готово) |
| POST | `/api/auth/login`    | публично | пароль → JWT |
| POST | `/api/order`         | касса    | занести заказ вручную → «готовится» |
| POST | `/api/order/status`  | касса    | сменить статус (`open`/`preparing`/`ready`/`served`) |
| POST | `/api/order/delete`  | касса    | удалить активный заказ |
| POST | `/api/day/reset`     | касса    | убрать активные с табло (выданные — в истории) |
| GET  | `/api/history`       | касса    | заказы за день с метками времени (`?date=`) |
| GET  | `/api/events`        | касса    | журнал событий за день (`?date=`) |
| GET  | `/api/stats/days`    | касса    | сводка по дням (заказов, ср. времена, день недели) |
| GET  | `/api/stats/range`   | касса    | сводка + по часам за выбранные дни (`?dates=d1,d2`) |
| GET  | `/api/backup/list`   | касса    | список бэкапов БД |
| GET  | `/api/backup/latest` | касса    | скачать последний бэкап (gzip) |
| GET  | `/api/qr?data=&border=` | публично | SVG QR-кода (для табло/наклейки) |
| GET  | `/api/health`        | публично | проверка живости |
